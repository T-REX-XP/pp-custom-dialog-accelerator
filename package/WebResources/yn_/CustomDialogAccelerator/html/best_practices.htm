Best Practices

This section contains best practices and common sense rules to build CustomPage or CanvasApp.

Common Rules:

  • Use naming convention: TBD
  • Localization:
    • Always tring to use resx files to store all text resources instead of the hardcoding them into the application.
    • Install Visual Studio Code addons to add/edit resx: ResX Editor
  • Make reusable components: Try to make generic components and extract them to a separate library to reuse them in different applications. It allows you to build the apps similarly: styling and components.

UI/UX:

  • Empty field placeholders:
    • Always use empty field placeholders like "---", to improve user exipience
    • Always use empty grid placeholders, to improve user exipience
